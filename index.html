述职文档参与业务

1.慢病需费卡

业务描述

慢病在好医生中算是一个较老的项目很早就有。可以为自己的家人朋友做档案，还可以咨询一流的专业医生去问诊，目前有以下服务如：慢病关爱卡、慢病安康卡、慢病关爱套餐、慢病无忧、慢病医生等服务，后续将开通鱼跃的服务。当用户开通这些服务之后重此用户就会有自己的慢病档案，实时监控自己身体的数据，第一时间了解自己的身体状况。

PS:慢病重构后的需求业务和以上所述一样

负责内容

参与慢病需费卡项目的开发。需费卡的需求是在原有的还没重构之前的的项目基础上添加新的需求，在这个过程中，我负责整个新需求的代码编写。框架以Backbone为基础架构，以grunt为开发构建工具。因为该项目在我初加入公司前就有，之前该项目的开发以离开公司，所以对项目之前的业务并不熟悉，当熟悉了解代码后需求完成。

PS: 在初期，除去完成一些为慢病填加topbar ，去cordova之类的小需求外，后面断断续续，一边进行项目的重构开发，一边进行老版慢病的新需求开发，以及修复老版本慢病以前的一些历史遗留bug。实际新框架的搭建是由一位资深的同事搭建，我负责具体业务以及逻辑的开发。

2. 慢病重构

业务描述

因为该项目是一个重构的项目( 业务需求是和以前一样的，这里就不重复提 )只是技术方案变化。

负责内容

公司内部需要统一技术栈，所以负责整个慢病项目的重构。新慢病以react＋express＋es6为基础架构，以 gulp为开发构建工具。重构的前提就是必须了解之前的业务，看懂之前的老代码，因为之前该项目的负责人都不在，我只能找该项目沾边的老同事那里获得一些信息，当我大体将老代码都弄的差不多后，马上就开始进行核心代码的开发。此后很长一段时间都在慢病重构的世界中度过。

PS: 在重构的最开始阶段只是单纯的重构慢病项目，随着时间的迁移～ 赶上啦慢病https升级、新的打点的改造beacon、以及新的去cordova改造。

最后挑出来一些产品觉得必须修复的bug修掉。

5. 对慢病重构完成的代码进行自我优化

业务描述

在手机app内已尽pc端进行测试，在进行优化。

负责内容

重构完成之后，有些页面因为逻辑深复杂，单张页面代码量超过1500行，太多太长，主要的原因是 react 的核心思想就是组件化，我将之前的一张页面划分为多个组件模块，但并未分文件。也将一些没必要分成2张页面的合并成2个组件最后合为一个页面。最后进行了取舍。

技术技能

在平安工作的这半年里，除了做业务之外，也结识了不少技术出众的同学，技术方面学习到了不少东西。

有一部分技术，原本接触过，会使用，但在来到我的团队后，我又对这些技术有了新的认识，拓展了视野，让我在这些技术上得到了巩固和提高。

巩固提高的技能

  • React + Redux + Router：异步路由，异步reducer，各种middleware，react性能优化等；
  • webpack + gulp：webpack、插件、loader，webpack-dev-server，gulp的各种插件等；
  • ES6 + Babel：babel的各种配置，es6的各种新语法(Symbol，async，promise，fetch等)；
  • Eslint：各类语法约束，规范编码风格。

还有一些技术，是过去没有接触过的，在这里学到了新知识。

新掌握的技能

  • Scss：过去只使用过less，在这里学会了一种新的预处理器；
  • CSS Module：将样式文件模块化，自动化地避免了样式污染；

团队协作

来到平安后，我加入了前端团队的寿险慢病小组。半年时间的合作下来，和各个同学也都混熟了。工作上现在也算合作的蛮顺利。无论是小组内经常合作的同学，还是前端团队内其他偶尔合作的同学，现在都能比较顺利的进行工作对接了。

总结

1、从刚接触react到现在已可以经熟练的进行react通用组件书写和需求开发。
2、从刚接触业务，到现在已基本了慢病业务。
3、感觉开发还是要详细了解业务需求，别太相信自己的记忆，要写好开发文档，把需求中的功能记在备忘录中，方便开发时查看和开发后核对。
4、做好代码复查，先进行快速查看，主要看看有没有明显的bug；然后主要进行逻辑测试，要用边界值，多角度，进行测试。尽量不要把bug，留到测试。
5、努力学习、做好总结，在技术这条路上越走越远。

平安是我的第二份工作，之前是在小型创业公司工作，来到这里让我体验了一个不一样的环境。总的来说，这儿气氛是比较好的，同事之间相处也挺融洽，而且有很多技术牛逼的同学，让我学习到了不少东西。最重要的是说，在我的团队中我感受到了浓厚的学术氛围，这对于一心走技术路线的我来说，是相当乐于见到的。这样的氛围，让每天的工作不仅只有做业务，也让我在不断学习，不断的提高自己。最后，希望我们的团队能一直保持下去，越走越远，也希望我自己能不断进步！
